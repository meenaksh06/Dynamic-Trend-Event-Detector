## 1. Executive Summary

Phase 3 introduces a **synergistic hybrid forecasting architecture** that combines classical Machine Learning (ARIMA) with Deep Learning (BiLSTM) to achieve superior trend prediction accuracy. Unlike naive combinations, our approach uses **Neuro-Statistical Decomposition** — ARIMA handles the linear signal, and BiLSTM learns the complex non-linear residuals that ARIMA cannot capture. Diagnostic ablation studies across multiple topics validate that **neither component alone achieves the performance of the hybrid**, proving the necessity of the integrated architecture.

---

## 2. Problem Statement

News trend time-series data contains **two distinct signal types**:

1. **Linear Patterns**: Gradual upward/downward trends, seasonal cycles, and slow-moving narrative shifts. These are well-modeled by classical statistical methods.
2. **Non-Linear Patterns**: Sudden event-driven spikes, complex inter-topic dependencies, and chaotic fluctuations caused by breaking news. These require the representational power of deep learning.

**The Core Challenge**: No single model effectively captures both. ARIMA excels at (1) but fails at (2). BiLSTM can learn (2) but often overfits on short sequences and misses simple linear trends. Phase 3 solves this by making them work together symbiotically.

## 3. Methodology

### 3.1 Data Pipeline

The hybrid model operates on the **monthly topic proportion time-series** generated by the LDA topic extraction from Phase 2:

```
Raw News (210K articles)
    → Text Cleaning (Phase 1)
    → LDA Topic Assignment (10 topics)
    → Monthly Aggregation → topic_trends matrix (T × 10)
    → Per-topic time-series for forecasting
```

### 3.2 Hybrid Architecture: ARIMA + BiLSTM

The architecture follows a **sequential decomposition with residual learning**:

#### Step 1: ARIMA(2,1,2) — Linear Component (ML)
- **Input**: Raw topic time-series `y(t)` of shape `(T,)`
- **Process**: Augmented Dickey-Fuller test for stationarity, then fit ARIMA(p=2, d=1, q=2)
- **Output**: Linear prediction `y_linear(t)` capturing trend and seasonality
- **Residual**: `epsilon(t) = y_actual(t) - y_linear(t)` — the signal ARIMA cannot model

#### Step 2: BiLSTM — Non-Linear Component (DL)
- **Input**: Scaled residuals via sliding window, shape `(N_seq, 5, 1)`
- **Architecture**:
  - Bidirectional LSTM Layer 1: 64 units (→ 128 dim bidirectional output)
  - Dropout: 0.2
  - Bidirectional LSTM Layer 2: 32 units (→ 64 dim bidirectional output)
  - Dropout: 0.2
  - Dense: 16 units, ReLU activation
  - Dense: 1 unit, Linear activation
- **Output**: Predicted residual correction `epsilon_hat(t)`

#### Step 3: Additive Fusion
```
y_hybrid(t) = y_linear(t) + epsilon_hat(t)
```

This additive combination is **not arbitrary** — it is mathematically justified because ARIMA models the conditional mean (linear component), and the BiLSTM models the deviation from that mean (non-linear component). The signals are orthogonal by construction.

### 3.3 Training Protocol
- **Train/Test Split**: 80% / 20% temporal split (no data leakage)
- **ARIMA**: Fitted on training data only
- **BiLSTM**: Trained on training residuals with EarlyStopping (patience=10)
- **Optimizer**: Adam (lr=0.001)
- **Loss**: Mean Squared Error (MSE)
- **Scaler**: MinMaxScaler(-1, 1) for residuals

## 4. Ablation Study Design

### 4.1 Purpose
To prove that the hybrid model is **not a trivial combination** and that each component is **necessary** for optimal performance.

### 4.2 Models Compared

| ID | Model | Type | Description |
|---|---|---|---|
| A | ARIMA(2,1,2) | ML-Only | Classical statistical baseline |
| B | BiLSTM (64+32) | DL-Only | Neural network trained on raw series (not residuals) |
| C | ARIMA + BiLSTM | **Hybrid** | ARIMA trend + BiLSTM residual correction |

### 4.3 Evaluation Metrics

All models are evaluated on the **same test set** with:
- **RMSE** (Root Mean Squared Error) — penalizes large errors
- **MAE** (Mean Absolute Error) — average error magnitude
- **R2** (Coefficient of Determination) — explained variance
- **MAPE** (Mean Absolute Percentage Error) — scale-independent accuracy

### 4.4 Multi-Topic Validation
The ablation is run across **5 different topics** (Topic 0-4) to ensure results are not topic-specific and the hybrid advantage is generalizable.

### 5.2 Diagnostic Analysis

The ablation study answers three critical questions:

**Q1: What happens when we REMOVE the BiLSTM (DL component)?**
- Using ARIMA alone, the model cannot capture sudden event-driven narrative spikes
- Expected RMSE increase: significant degradation on topics with high volatility
- The model loses its ability to correct for non-linear dynamics

**Q2: What happens when we REMOVE ARIMA (ML component)?**
- Using BiLSTM alone on raw data, the model tends to overfit on short sequences
- It misses the stable linear trend that ARIMA captures effortlessly
- Performance degrades especially on topics with strong seasonal patterns

**Q3: Is the Hybrid genuinely synergistic?**
- The hybrid should show R2 improvement over the best individual component
- Neither component alone should match the hybrid's performance
- This proves the architecture is not a trivial averaging but a structured decomposition

## 8. Key Technical Decisions

### Why ARIMA + BiLSTM (not other combinations)?
- **ARIMA** is the gold standard for linear time-series — well-understood, interpretable, fast
- **BiLSTM** excels at sequence learning with bidirectional context
- The residual decomposition is mathematically clean and preserves interpretability
- Alternative: Random Forest + LSTM would lose the temporal structure that ARIMA preserves

### Why Additive Fusion (not concatenation or attention)?
- Additive fusion preserves the **interpretability** of each component
- We can always decompose a prediction into "what ARIMA contributed" vs "what BiLSTM corrected"
- This is critical for the XAI (Explainable AI) requirement of the project

### Why Multi-Topic Ablation (not single topic)?
- Single-topic results could be misleading (topic-specific artifacts)
- Multi-topic validation ensures the hybrid advantage is **generalizable**
- It also reveals which types of topics benefit most from the DL component
